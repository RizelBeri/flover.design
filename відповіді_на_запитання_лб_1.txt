На які дві частини ділиться процес web-розробки?
    Frontend – все, що бачить і з чим взаємодіє користувач (інтерфейс, верстка, стилі, анімації, логіка на боці клієнта).

    Backend – серверна частина, що відповідає за обробку запитів, збереження й даних, логіку роботи сайту.

В чому полягають завдання front-end розробника?
    - Створення та верстка інтерфейсу (HTML, CSS).

    - Додавання інтерактивності за допомогою JavaScript.

    - Адаптація сайту під різні пристрої й браузери.

Яка функція браузеру у процесі взаємодії користувача із web-сервером?
    - відправляє HTTP/HTTPS-запит на сервер,
     
    - отримує у відповідь HTML, CSS, JS, зображення та інші файли,
     
    - інтерпретує й відображає їх у вигляді веб-сторінки для користувача.

Які основні елементи включає структура web-браузеру?
    - User Interfase (адресний рядок, вкладки, кнопки назад/вперед тощо).
- 
    - Browser engine (керує взаємодією між інтерфейсом і рушієм рендерингу).
    
    - Рендеринг-рушій (перетворює HTML/CSS/JS у відображення на екрані).
     
    - Інтерпретатор JavaScript (наприклад, V8 у Chrome).

    - Система зберігання даних (cookies, localStorage, cache).

    - Мережовий модуль (обробка http та https).

В чому полягає схема роботи web-браузеру?
    - Користувач вводить URL або клікає на посилання.
 
    - Браузер через DNS дізнається IP-адресу сервера.
 
    - Встановлюється з’єднання з сервером (TCP/IP, HTTPS).

    - Надсилається запит на потрібний ресурс.
 
    - Сервер повертає відповідь (HTML, CSS, JS, медіа).

    - Браузер завантажує й рендерить сторінку.

З чого складається URL-адреса?
    - Протокол – http:// або https://.
 
    - Доменне ім’я – наприклад, example.com.
     
    - Порт (опціонально).
     
    - Шлях до ресурсу – /page/index.html.
    
    - Параметри запиту 

Яку роль грає система доменних імен у процесі роботи web-браузеру?
    Перетворює зрозумілу для людини адресу (example.com) на IP-адресу сервера (xx.xxx.xxx.xx).

Для чого було введено семантичну розмітку у HTML5?
    Для структурування контенту за його призначенням, що полегшує розуміння сторінки як для пошукових систем, так і для користувачів,
